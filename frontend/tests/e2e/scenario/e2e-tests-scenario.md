# 페이지별 테스트 시나리오

## 🎯 테스트 목적 및 범위

### 주요 목적
1. **사용자 경험 검증**: 실제 사용자 관점에서의 플랫폼 사용성 확인
2. **기능 통합 검증**: 개별 기능들이 전체 워크플로우에서 올바르게 동작하는지 확인
3. **실시간 시스템 안정성**: WebSocket 기반 실시간 데이터 시스템의 안정성 검증
4. **비즈니스 로직 정확성**: 투자 및 거래 관련 핵심 비즈니스 로직 검증

### 테스트 범위
- ✅ 메인 페이지 (`/`)
- ✅ 사용자 인증 시스템 (`/signup`, `/signin`, `/reset-password`, 네브바 닉네임 변경, UX)
- ✅ 실시간 거래소 (`/exchange`)
- ✅ 포트폴리오 추천 시스템 (`/portfolio-pilot`)
- ✅ 시장 트렌드 분석 (`/trends`)
- ✅ 사용자 자산 관리 (`/my-assets`, `/history`)

### 우선순위 정의
- **Critical**: 플랫폼 핵심 기능, 실패 시 서비스 불가
- **High**: 주요 사용자 기능, 실패 시 사용자 경험 크게 저하
- **Medium**: 부가 기능, 실패 시 제한적 영향

## 🚀 테스트 실행 전략

### 사전 준비
```bash
# 프론트엔드 실행
pnpm dev

# WebSocket 서버 실행 (별도 터미널)
cd ../backend && npm run dev

```

### 핵심 사용자 여정 (Critical Path)
1. **신규 사용자 온보딩**:
   - 회원가입 → 이메일 인증 → 로그인 → 거래소 접근
2. **기본 거래 플로우**:
   - 마켓 선택 → 실시간 데이터 확인 → 매수 주문 → 체결 확인
   - 마켓 선택 → 실시간 데이터 확인 → 매도 주문 → 체결 확인
3. **포트폴리오 파일럿**:
   - 전략 선택 → 투자금액 설정 → 구성된 포트폴리오로 매수 → 보유 자산 페이지로 이동해서 확인

### 테스트 실행 명령어
```bash
# 전체 E2E 테스트 실행
pnpm test:playwright

# 특정 파일만 테스트 실행
pnpm test:playwright $ARGUMENT
```
